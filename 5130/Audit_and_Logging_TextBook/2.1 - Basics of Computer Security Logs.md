# 2.1 — The Basics of Computer Security Logs

> **Core Idea:** A log is a record of events that occur within an organization's systems and networks. Security logs are generated by many sources — security software, operating systems, and applications — and they serve as the foundation for monitoring, incident detection, forensics, and compliance.

---

## 📝 What Is a Log?

A **log** is a record of events occurring within an organization's systems and networks. Each log entry typically contains:

| Field | Description | Example |
|-------|-------------|---------|
| **Timestamp** | When the event occurred | 2026-04-14T03:22:15Z |
| **Source** | The system, application, or device that generated the entry | Firewall, Apache server, Windows OS |
| **Event type / Category** | Classification of what happened | Authentication, authorization, system event, error |
| **Severity / Priority** | How important or critical the event is | Emergency, Alert, Critical, Error, Warning, Notice, Info, Debug |
| **User identity** | Who triggered the event (if applicable) | Username, user ID, IP address |
| **Details / Message** | Description of what happened | "Failed login attempt for user admin from 192.168.1.105" |
| **Outcome** | Success or failure of the action | Success, Failure, Denied |

---

## 🛡️ 2.1.1 — Security Software

Security software generates some of the **most valuable** logs for detecting and investigating threats.

### Types of Security Software That Generate Logs

| Software Type | What It Logs | Example Entries |
|--------------|-------------|-----------------|
| **Antivirus / Antimalware** | Malware detections, quarantine actions, scan results, signature updates | "Trojan.GenericKD detected in C:\Users\downloads\file.exe — quarantined" |
| **IDS / IPS** (Intrusion Detection / Prevention Systems) | Detected attack signatures, anomalous behavior, blocked attacks | "SQL injection attempt detected from 10.0.0.55 on port 443 — blocked" |
| **Firewalls** | Allowed and denied connections, rule matches, traffic volume | "DENY TCP 203.0.113.5:54321 → 192.168.1.10:22 (rule: block-external-SSH)" |
| **VPN Gateways** | Remote access connections, authentication events, session duration | "User jsmith connected via VPN from 98.45.67.89 at 08:15; session 4h 22m" |
| **Authentication servers** | Login attempts (success/failure), account lockouts, password changes | "Authentication failure for admin — 5th consecutive attempt — account locked" |
| **Proxy servers / Web filters** | URLs accessed, categories blocked, bandwidth usage | "BLOCK: user01 attempted access to gambling site (policy: restricted-categories)" |
| **DLP (Data Loss Prevention)** | Sensitive data transfers, policy violations, blocked exfiltration | "ALERT: SSN pattern detected in outbound email attachment — blocked" |
| **Vulnerability scanners** | Discovered vulnerabilities, missing patches, misconfigurations | "Critical: CVE-2025-1234 (CVSS 9.8) found on server web-prod-01" |

---

## 💻 2.1.2 — Operating Systems

Operating systems generate logs about **system-level events** that are critical for both security monitoring and troubleshooting.

### What OS Logs Capture

| Category | Events Logged | Security Relevance |
|----------|--------------|-------------------|
| **Authentication / Logon events** | Successful and failed logins, logoff events, account lockouts | Detecting brute force attacks, unauthorized access, unusual login times/locations |
| **Account management** | Account creation, deletion, modification, privilege changes | Detecting unauthorized account creation, privilege escalation |
| **Process / Application execution** | Programs started, stopped, or crashed | Detecting unauthorized software, malware execution |
| **System events** | Startup, shutdown, service starts/stops, driver loads | Detecting unauthorized reboots, service tampering |
| **Audit policy changes** | Changes to logging configuration itself | Critical — attackers often disable logging first |
| **Object access** | File access, registry access, directory access | Detecting unauthorized data access, data exfiltration |
| **Privilege use** | Use of elevated/special privileges | Detecting privilege abuse |

### Platform-Specific Log Systems

| OS | Log System | Key Logs |
|----|-----------|----------|
| **Windows** | Windows Event Log (evtx) | Security, System, Application, Setup, Forwarded Events |
| **Linux / Unix** | Syslog, journald, auth.log | /var/log/auth.log, /var/log/syslog, /var/log/messages, /var/log/kern.log |
| **macOS** | Unified Logging (os_log) | system.log, install.log, security audit logs |

---

## 📱 2.1.3 — Applications

Applications generate logs specific to their function — these provide **context** that OS and security logs may lack.

### Application Log Sources

| Application Type | What It Logs | Example |
|-----------------|-------------|---------|
| **Web servers** (Apache, Nginx, IIS) | HTTP requests, response codes, client IPs, URLs, user agents | "GET /admin/login HTTP/1.1 200 — from 45.33.32.0 — Mozilla/5.0" |
| **Database servers** (MySQL, PostgreSQL, Oracle) | Queries executed, authentication, schema changes, errors | "WARN: user 'app_user' executed DROP TABLE customers" |
| **Email servers** (Exchange, Postfix) | Messages sent/received, sender/recipient, attachment info, spam filtering | "REJECT: message from suspicious-domain.ru — spam score 9.8/10" |
| **DHCP servers** | IP address assignments, lease renewals, MAC-to-IP mappings | "ASSIGN: 192.168.1.50 to MAC 00:1A:2B:3C:4D:5E — lease 24h" |
| **DNS servers** | Name resolution queries and responses | "QUERY: client 10.0.0.15 requested A record for malware-c2.evil.com" |
| **Custom / Business applications** | Application-specific actions, transactions, errors | "ORDER #12345 placed by user_jane — $2,340 — payment approved" |

## ✅ 2.1.4 — Usefulness of Logs

### Why Logs Matter

Logs serve multiple critical purposes across the organization:

| Purpose | How Logs Help |
|---------|--------------|
| **Incident detection and response** | Detect attacks in progress; provide timeline of events for investigation; correlate events across systems |
| **Forensic analysis** | Reconstruct what happened after an incident; determine scope and impact; identify attacker techniques |
| **Compliance and auditing** | Prove adherence to regulations (HIPAA, PCI-DSS, SOX, GDPR); provide audit trails for internal and external auditors |
| **Troubleshooting** | Diagnose system errors, performance issues, and misconfigurations |
| **Baseline establishment** | Establish normal behavior patterns so anomalies can be detected |
| **Accountability** | Link actions to specific users for non-repudiation; deter misuse through awareness of monitoring |
| **Trend analysis** | Identify patterns over time — increasing attack frequency, recurring errors, peak usage periods |
| **Legal evidence** | Logs can serve as evidence in legal proceedings if properly collected and preserved (chain of custody) |

### Log Usefulness Depends On:
- **Completeness** — are all relevant events being logged?
- **Accuracy** — are timestamps correct and synchronized?
- **Integrity** — have logs been tampered with?
- **Retention** — are logs kept long enough for investigation and compliance?
- **Accessibility** — can the right people find and analyze logs when needed?

---

## 💡 Key Takeaways

> [!tip] Remember
> - Logs come from **three main sources**: security software, operating systems, and applications
> - Each source provides different but complementary information — **correlation across sources** is key
> - Logs support **six major functions**: detection, forensics, compliance, troubleshooting, accountability, and legal evidence
> - Log usefulness depends on **completeness, accuracy, integrity, retention, and accessibility**
> - **Firewalls, IDS/IPS, and authentication logs** are among the most security-critical
> - Attackers often **disable or tamper with logging** as a first step — protecting log integrity is essential
