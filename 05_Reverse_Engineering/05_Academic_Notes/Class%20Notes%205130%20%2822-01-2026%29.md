# Class Notes 5130 (22-01-2026)

## Security vs. Performance
- **Secure Functions**: Often 2x-3x slower than unsafe ones.
- **Canaries**: Provide excellent security with only a ~3% performance overhead, making them a standard production choice.

---

## 🛡️ Stack Canaries (Compile-Time)
Compilers inject code at function entry and exit to check for corruption.
- **Deterministic**: Null canaries, Terminator canaries (easier to predict).
- **Indeterministic**: Random canaries (decided at compile-time), XOR canaries (changes on every execution).

---

## 🔒 Modern Defenses
- **NX (No-Execute / DEP)**: Prevents execution from segments like the stack. Also known as **W xor X** (Writable or Executable, but not both).
- **ASLR**: Randomizes base addresses of Heap and Stack (but not .text).
- **PIE**: Extends randomization to the **.text** (code) section.
- **RELRO**: Protects the PLT/GOT from being overwritten by making it read-only after startup.
